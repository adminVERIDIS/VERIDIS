# Script Call Decouverte VERIDIS

Objectif : comprendre la situation, etablir la confiance, creer une urgence douce et verifier si VERIDIS est pertinent.

Duree : 30 minutes.

Posture : consultant bienveillant. Pas de pression. Pas de demo generique. Pas de discount improvise.

## Avant le call

Preparation en 5 minutes :

- verifier role, entreprise, secteur, effectif ;
- regarder le site et un eventuel rapport RSE ;
- noter 1 hypothese de douleur ;
- ouvrir la fiche Notion ;
- preparer un rapport sectoriel ou exemple proche ;
- verifier le plan probable : Essential, Professional ou Enterprise.

## 00:00-02:00 - Rapport

Script :

> Bonjour {{prenom}}, merci d'avoir pris le temps.
>
> Avant de parler CSRD, comment allez-vous ? C'est le feu en ce moment avec les echeances et les sujets reporting ?

Objectif :

- detendre ;
- comprendre l'energie du prospect ;
- ne pas vendre trop tot.

Note CRM :

- humeur ;
- contexte ;
- urgence visible.

## 02:00-07:00 - Contexte

Script :

> Racontez-moi : ou en etes-vous sur la CSRD aujourd'hui ?
>
> Qui porte le sujet dans l'entreprise ?
>
> Est-ce que vous avez deja un rapport RSE, un brouillon CSRD ou une premiere analyse de materialite ?

Relances utiles :

- "Qu'est-ce qui est deja en place ?"
- "Qu'est-ce qui manque encore ?"
- "Qui doit valider a la fin ?"
- "Vous travaillez avec un cabinet, en interne, ou hybride ?"

Ecoute : 80%.

Note CRM :

- stage du projet ;
- decisionnaire ;
- parties prenantes ;
- documents disponibles.

## 07:00-17:00 - Problemes

Script :

> Qu'est-ce qui vous prend le plus de temps en ce moment ?
>
> Si vous deviez prioriser une chose avant l'echeance, ce serait quoi ?
>
> Qu'est-ce qui vous empeche de dormir tranquillement sur ce sujet ?

Relances :

- "Pourquoi celui-la en premier ?"
- "Qu'est-ce qui bloque concretement ?"
- "C'est plutot un probleme de donnees, de methode, de gouvernance ou de temps ?"
- "Comment vous savez aujourd'hui si votre dossier est pret, partiel ou fragile ?"

Objectif :

- identifier 2-3 douleurs precises ;
- eviter de parler features ;
- comprendre le cout du probleme.

Note CRM :

- douleur principale ;
- douleurs secondaires ;
- mots exacts du prospect.

## 17:00-22:00 - Impact

Script :

> Si vous ne resolvez pas {{douleur_principale}}, qu'est-ce qui se passe ?
>
> Et au niveau comite de direction, audit, clients ou financement, quels sont les risques ?

Relances :

- "Qui serait impacte en interne ?"
- "Combien de temps l'equipe passe dessus aujourd'hui ?"
- "Quel serait le cout d'un retard ou d'une correction tardive ?"
- "Qu'est-ce qui changerait si vous aviez une lecture claire cette semaine ?"

Objectif :

- faire verbaliser la valeur par le prospect ;
- qualifier urgence et budget ;
- eviter l'urgence artificielle.

Note CRM :

- impact business ;
- timing ;
- budget ou blocage budget ;
- authority.

## 22:00-27:00 - Solution implicite

Script :

> Si vous aviez un outil qui analyse votre rapport, sort un score de conformite, identifie les gaps et vous donne les 3 priorites, ca changerait quoi pour vous ?

Silence. Laisser le prospect repondre.

Si reponse positive :

> Ce que vous decrivez, c'est exactement pourquoi j'ai construit VERIDIS.
>
> Je peux vous montrer ce que ca donne sur un cas proche de votre secteur ?

Si reponse floue :

> J'ai peut-etre mal compris le besoin. Qu'est-ce qui serait vraiment utile pour vous dans les 30 prochains jours ?

Objectif :

- obtenir permission de demo ;
- valider que VERIDIS resout une douleur reelle ;
- ne pas forcer.

## 27:00-30:00 - Transition et prochaine etape

Option A - Demo maintenant :

> J'ai un exemple pret. Je vous propose de prendre 10 minutes pour vous montrer le diagnostic, puis on decide ensemble si ca vaut le coup d'aller plus loin.

Option B - Demo separee :

> Le plus utile serait un call demo de 20 minutes avec un exemple proche de votre secteur. Jeudi 14h ou vendredi 10h ?

Option C - Pas qualifie :

> Je pense que ce n'est pas le bon moment pour vous vendre VERIDIS. Je vous propose plutot de vous envoyer notre checklist CSRD et de reprendre dans 30 jours.

## BANT rapide pendant ou apres le call

| Critere | Question | Positif si |
| --- | --- | --- |
| Budget | Avez-vous un budget alloue pour la conformite CSRD ? | Budget existe ou peut etre arbitre |
| Authority | Etes-vous la personne qui porte ou decide le sujet ? | DAF, dirigeant, RSE owner ou acces decisionnaire |
| Need | Avez-vous deja commence votre rapport ou diagnostic ? | Douleur explicite |
| Timing | Quelle est votre echeance ou prochaine revue ? | Besoin dans les 90 jours |

Decision :

- BANT 4/4 : demo et proposition possible.
- BANT 3/4 : demo + clarifier le point faible.
- BANT 2/4 : nurture ou diagnostic gratuit leger.
- BANT 0-1/4 : newsletter + relance 30 jours.

## Fin de call obligatoire

Ne jamais raccrocher sans :

- prochaine action ;
- date ;
- responsable ;
- note CRM ;
- statut pipeline.

Phrase :

> Pour etre clair, la prochaine etape est {{action}}, c'est moi qui m'en occupe, et je vous l'envoie avant {{date}}. C'est bon pour vous ?

## Commentaire apres script

Le call decouverte n'est pas une scene de vente. C'est un diagnostic. Si le prospect repart avec plus de clarte meme sans acheter, VERIDIS gagne en confiance.
