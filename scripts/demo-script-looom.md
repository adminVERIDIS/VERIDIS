# Script Demo Loom VERIDIS

Duree : 8-10 minutes.

Public : prospect interesse mais indisponible pour un call live.

Objectif : montrer la valeur de VERIDIS de facon personnalisee, puis pousser vers Calendly ou Stripe selon maturite.

## Preparation

- Choisir un rapport public ou anonymise proche du secteur du prospect.
- Preparer le dashboard VERIDIS deja connecte.
- Masquer toute donnee sensible.
- Ouvrir la fiche Notion du prospect.
- Noter les 2 douleurs mentionnees par le prospect.
- Preparer le lien Calendly et le lien Stripe du plan recommande.

## Structure

### 00:00-00:45 - Intro personnalisee

> Bonjour {{prenom}},
>
> Vous m'avez dit que vous n'aviez pas le temps pour un call cette semaine. Je comprends : les sujets CSRD prennent vite beaucoup de place.
>
> Du coup, j'ai prepare cette demo personnalisee pour {{entreprise}}. En moins de 10 minutes, vous allez voir ce que VERIDIS detecte sur un rapport du secteur {{secteur}}.

### 00:45-01:30 - Probleme

> Le point que vous avez mentionne, c'est {{douleur_principale}}.
>
> Je ne vais donc pas faire une demo generale. Je vais vous montrer comment VERIDIS aide a repondre a trois questions :
>
> 1. est-ce que le dossier est conforme, partiel ou expose ?
> 2. quels gaps sont prioritaires ?
> 3. quelle action lancer cette semaine ?

### 01:30-03:00 - Upload et analyse

> Je prends ici un rapport RSE proche de votre secteur.
>
> Je lance l'analyse. L'objectif n'est pas de remplacer un auditeur ou un consultant, mais de produire une premiere lecture robuste en quelques minutes.

Action ecran :

- upload fichier ;
- attendre/animer ;
- montrer que l'analyse est structuree.

### 03:00-05:00 - Score

> Voila le resultat.
>
> Score estime : {{score}}/100.
>
> La categorie est {{categorie}}. Cela veut dire qu'il y a {{interpretation_score}}.

Silence 3 secondes.

> Ce qui compte ici, ce n'est pas seulement le chiffre. C'est l'explication : pourquoi le score est comme ca, et quels elements doivent etre corriges.

### 05:00-07:30 - Gaps prioritaires

> VERIDIS remonte ici les trois priorites.

1. {{gap_1}} - impact : {{impact_1}}
2. {{gap_2}} - impact : {{impact_2}}
3. {{gap_3}} - impact : {{impact_3}}

> Je clique sur le premier gap. Vous voyez le guide pas a pas : ce qui manque, pourquoi c'est important, quelle preuve ajouter, et quelle action lancer.

Action ecran :

- cliquer un gap ;
- montrer le guide ;
- montrer le proprietaire/action/date si disponible.

### 07:30-08:45 - Rapport et decision

> Ensuite, vous pouvez exporter un rapport clair pour votre comite, votre equipe finance/RSE ou votre consultant.
>
> L'interet pour vous : ne pas passer deux semaines a chercher ou commencer. Vous avez un score, des gaps et une priorite.

### 08:45-10:00 - CTA

Option A - prospect chaud :

> Vu votre contexte, je recommande le plan {{plan_recommande}}.
>
> Si vous voulez demarrer, voici le lien : {{stripe_checkout}}.
>
> Vous avez 14 jours pour tester. Si VERIDIS ne vous apporte pas de clarte, je vous rembourse.

Option B - besoin de discussion :

> Si vous voulez qu'on regarde votre cas ensemble, prenez un creneau ici : {{calendly}}.
>
> En 20 minutes, on peut refaire l'exercice avec votre contexte et voir si VERIDIS est pertinent.

Outro :

> Merci {{prenom}}. Repondez simplement a cet email avec vos questions, je vous repondrai personnellement.
>
> A bientot,
>
> {{signature}}

## Commentaire apres script

Une demo Loom doit etre plus courte qu'un call live. Si elle depasse 10 minutes, couper les features et garder seulement probleme, score, gaps, prochaine etape.
